    2^n mod 5¶

    分析¶

    设当前输入为B，当前状态为x：x=2^{B}\ \rm mod\ 5，下一次输入一位b(b=0,1)，则可得**次态**x'为： $$ x'=2^{B'} \rm mod 5\tag{1} $$ 又由串行输入的规律（从高位输入，每输入一个数字就相当于之前输入的数左移一位再加上当前输入的数字），易得： $$ B'=2B+b\tag{2} $$ 其中B又可以表示为： $$ B=log_2(5k+x),k=0,1,2,\dots\tag{3} $$ 将(2)和(3)代入(1)可得： $$ \begin{aligned} x'&=2^{2log_2(5k+x)+b} {\rm mod} 5 \&=(5k+x)^2\cdot 2^b {\rm mod} 5 \&=(25k2+10kx+x2)\cdot 2^b {\rm mod} 5 \&=x^2\cdot 2^b {\rm mod} 5 \end{aligned} $$ 综上，x可以取\{1,2,3,4\}，共4种状态，考虑用2位二进制编码（S1和S0）表示状态。

    状态转移¶

    我们以周期为**时间段的单位计时。方便起见，这里将最先到来的周期记为第 0 周期。 **每个周期内不同时间点的输入可能变化，但每个周期内的状态一定不变。

    状态转移行为可以描述成如下过程：

    • 当第 0 周期开始，状态state设为初始状态state_0 。
    • 每当第 n 周期结束前的最后一瞬间，记该瞬间的输入为 input_n 。
    • 每当第 n 周期结束，第n+1 周期开始时，状态变为状态转移函数给出的次态：

    $$ state_{n+1}=F_{next state}(state_n,input_n) $$ 注意：这里state_n和input_n是有下标n的，亦即state_{n}随周期数变化

    在此题为： $$ 状态转移函数/映射f:\begin{Bmatrix}状态集合x{1,2,3,4}\输入集合b{0,1}\end{Bmatrix}\rightarrow 状态集合x'(同x){1,2,3,4} $$ 具体转移函数/映射关系如下图所示：

    输出函数¶

    在**任意周期中的任意时间点**，输出为 $$ output=F_{output}(state,input) $$ 这里的 input 是当前时间点的输入， state 是当前时间点的状态。

    注意：这里state和input是没有下标n的，亦即output与周期数无直接关系

    区分状态机类型时：当输出函数的结果会随 input 变化而改变时，该状态机为 Mealy 机，否则为 Moore 机。

    题目要求即使在时钟上沿没有到来的时候，如果输入变化，输出也必须随其变化，也就是说题目要求搭建**Mealy型状态机**，输出由输入和当前状态共同决定。

    结合状态转移图，可得出 状态转移（组合逻辑） 和 输出电路（组合逻辑）的真值表（Truth Table）：

    0 1 1 1 0 s_3
    1 0 0 1 1 s_4
    1 0 1 1 0 s_3
    1 1 0 0 0 s_1

    Logisim_fibonacci_2¶

    算法分析¶

    快速幂¶

    快速幂算法是一种用于高效计算一个数的幂次的优化方法。它基于指数的二进制表示，通过将指数分解成二进制位来减少计算次数。

    考虑计算 a^n，其中 a 是底数，n 是指数。将指数 n 表示为二进制形式（n_i\in\{0,1\},i=0,1,2,\dots,t）： $$ n=(n_t\dots n_1n_0)_{\rm b}=n_t\times 2^t+\dots+n_1\times 2^1+n_0\times 2^0 $$

    则计算 a^n 可以简化为： $$ an=a}=a{n_t2t}\times\dots\times a{n_121}\times a{n_020} $$

    递归形式¶

    快速幂的递归算法采用分治策略，将问题拆分成较小的子问题。 $$ \begin{matrix}{\rm power}(a,n)= \begin{cases} {\rm power}(a,\frac{n}{2})\times{\rm power}(a,\frac{n}{2})&,n为偶数且n\neq 0 \ {\rm power}(a,\lfloor \frac{n}{2}\rfloor)\times{\rm power}(a,\lfloor\frac{n}{2}\rfloor)\times a&,n为奇数 \0&,n=0 \end{cases}\end{matrix} $$

    代码：

    int power(int a, int n) {
    if (n == 0) return 1;
    return (n & 1) ? power(a, n >> 1)*power(a, n >> 1)*a:power(a, n >> 1)*power(a, n >> 1);
}

    迭代形式¶

    显然我们很难让电路去实现递归，因此考虑迭代形式。

    迭代形式中，我们从指数的二进制表示的最低位开始，如果该位为 1，就将结果乘以当前的底数。然后，将底数自身平方，指数除以 2。这个过程一直进行，直到指数为 0。

    代码：

    int power(int a, int n) {
    int res = 1; // 存储最终结果 
    while (n > 0) { // 依次扫描 n 的每个二进制位直到 n = 0 
        if (n & 1) res *= a; // 如果当前二进制位为 1，则将乘子当前值乘入最终结果 
        a *= a; // 乘子更新为自身的平方 
        n >>= 1; // n 向右移位，扫描下一位 
    }
    return res;
}

    斐波那契数列的矩阵递推¶

    参考1：OI Wiki

    参考2：走肖暂时无法接通

    参考3：ChatGPT-3.5

    由斐波那契数列的递推性质可得： $$ F(n+1) = F(n) + F(n-1) $$

    F(n) = F(n)

    其中F(n) 表示斐波那契数列的第 n 项。通过矩阵的乘法，我们可以得到矩阵递推公式为： $$ \begin{bmatrix} F(n+1) & F(n) \ F(n) & F(n-1) \end{bmatrix} = \begin{bmatrix} 1 & 1 \ 1 & 0 \end{bmatrix} \cdot \begin{bmatrix} F(n) & F(n-1) \ F(n-1) & F(n-2) \end{bmatrix} $$ 记为： $$ M(n)=A\cdot M(n-1) $$ 这一版本的递推公式

    • 将加法转化为乘法
    • 将二阶递推式转化为一阶递推式
    • 将加和关系转化为等比数列

    从而简化了计算过程。于是我们十分容易地得到通项如下： $$ M(n)=A^n\cdot M(0) $$ 进一步分析，设 $$ A^n=\begin{bmatrix} 1 & 1 \ 1 & 0 \end{bmatrix}^n=\begin{bmatrix} a & b \ c & d \end{bmatrix} $$ 因为 $$ \begin{matrix}\begin{aligned} \begin{bmatrix} F(n+1) & F(n) \ F(n) & F(n-1) \end{bmatrix} &= \begin{bmatrix} 1 & 1 \ 1 & 0 \end{bmatrix}^n \cdot \begin{bmatrix} F(1) & F(0) \ F(0) & F(-1) \end{bmatrix} \&= \begin{bmatrix} 1 & 1 \ 1 & 0 \end{bmatrix}^n \cdot \begin{bmatrix} 1 & 0 \ 0 & 1 \end{bmatrix} \&= \begin{bmatrix} a & b \ c & d \end{bmatrix} \cdot \begin{bmatrix} 1 & 0 \ 0 & 1 \end{bmatrix} \&=\begin{bmatrix} a & b \ c & d \end{bmatrix} \end{aligned}\end{matrix} $$

    其中F(-1)严格意义上不存在，但不妨碍我们在这里求出它的值为F(-1)=F(1)-F(0)=1

    通过取矩阵 A^n 的**右上角**元素 b，即可得到第 n 项斐波那契数列的值。因此，计算 F(n) 的问题转化为矩阵 A^n 的求幂问题。

    那么数的快速幂可不可以**类比**运用到矩阵上呢？答案是肯定的。

    模块设计¶

    这个“数\rightarrow矩阵”的类比体现在元器件上的话，就是我们要专门设计针对矩阵的模块：

    • 普通寄存器只能储存一个**数**，因此我们构建一个针对**矩阵**的模块矩阵寄存器

    为了和普通寄存器功能一致，接入三个接口：

    • ENA
    • CLK

    • CLR

    • 普通MUX只能在两个**数**间选择，因此我们构建一个针对**矩阵**的模块矩阵MUX

    为了和普通MUX功能一致，接入一个接口：

    • select

    • 普通计算器Mul只能在两个**数**之间做乘法，因此我们构建一个针对**矩阵**的模块矩阵Mul

    核心设计¶

    三个寄存器的设置、初始化及其迭代过程：

    • A（矩阵寄存器）
    • 初始化：\begin{bmatrix}1 & 1 \\1 & 0\end{bmatrix}；
    • 迭代：A:=A\cdot A
    • S（矩阵寄存器）
    • 初始化：\begin{bmatrix}1 & 0 \\0 & 1\end{bmatrix}（同单位阵E）;

    • 迭代：S:=\begin{cases}E\cdot S&,n_0=0\\A\cdot S&,n_0=1\end{cases}，其中n_0为指数n=(n_t\dots n_1n_0)_{\rm b}的最低位

    • n（普通寄存器）

    • 初始化：input（输入值N）；
    • 迭代：n:=n>>1

    注：所谓的**初始化**并不是寄存器从一开始就存储了某个值，而是第1个上升沿（从T=0到T=1）到来时为寄存器赋值，这才完成初始化。

    其它细节设计¶

    • 寄存器冻结条件：

    • 当 n=0 时（某个上升沿），使得enable使能信号由1变0，寄存器不再改变（当前这个上升沿仍会赋值，下个上升沿开始则不再赋值）（这是因为当前这个enable信号的改变就是**这个**上升沿赋值后的**结果**，换句话说enable信号由1变0时，**这个**上升沿已经结束了）

    • 由于寄存器本来的值为0，会使得第0个周期enable信号为0（我们希望的是n迭代为0之前enable一直为1），因此给enable信号**初始化**为1

    • 输出条件

    • 第1个上升沿初始化；再经过32个上升沿，32bit的的输入n即可处理完毕

    • 当counter \geq3 3/(21)_{16} 时输出s_{12}（这里保险起见，设为了严格>34/(22)_{16}）
    • 在此之前一直输出0
